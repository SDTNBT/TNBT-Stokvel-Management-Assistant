# Sprint2 Requirements

# Objectives

1. Define Sprint 2 functional requirements (user stories)
2. Establish pre-Sprint 2 action items
3. Document role definitions and deadlines

---

## Meet up with Client

The team must meet with the client once per week. This is mandatory and does not count towards the four internal meetings the team must have.

**Sprint 2 Context:**

Following the completion of Sprint 1, the team reviewed feedback, identified mistakes, and outlined key requirements before proceeding to Sprint 2. The team met with the client to clarify expectations and ensure alignment with project goals.

---

## Choose Specifications

**Sprint 2 Functional Requirements (User Stories):**

The team agreed on the following user stories for Sprint 2:

| # | Role | User Story |
|---|------|-------------|
| 1 | Admin | As an Admin, I want to view all members in a group so that I can manage them |
| 2 | Member | As a Member, I want to view my profile so that I can see my details |
| 3 | Admin | As an Admin, I want to schedule meetings so that members are informed about upcoming events |
| 4 | Admin | As an Admin, I want to post meeting agendas so that members know what will be discussed |
| 5 | Admin | As an Admin, I want to record meeting minutes so that there is a record of decisions made |
| 6 | Admin | As an Admin, I want to remove member(s) from the group so that I can manage group membership |
| 7 | Member | As a Member, I want to receive notifications so that I stay updated on meetings and activities |

**Pre-Sprint 2 Requirements (Action Items):**

| Requirement | Description |
|-------------|-------------|
| Alternative Login | Implement a third-party authentication option (e.g., Google login) |
| Code Coverage | Improve test coverage to at least 50%. Testing was only set up during Sprint 1 and needs expansion |
| Deployment Issues | Identify and resolve existing deployment problems |

**Task Allocation:**

- Each team member selected one user story to work on for Sprint 2
- Members will be responsible for both frontend and backend implementation of their chosen user story

**Deadlines:**

- All members must ensure that any issues from Sprint 1 related to their work are fixed
- Previously non-functional features must be corrected
- Deadline for Sprint 2 completion: Sunday morning
